2.2 Creación de puntos de variación

Top  Previous  Next

Para la creación de un punto de variación lo primero que se debe hacer es crear un proceso en el Core Process, situandonos dentro de la carpeta de capacidad o de entrega. Para ello haremos Click con el botón izquierdo sobre la carpeta de capacidad por ejemplo y generamos un patrón de capacidad (Capability Pattern).

Una vez generado el proceso se nos abrirá un editor que usaremos para trabajar sobre el proceso seleccionado. En el proceso generado podremos crear cualquier elemento SPEM al igual que se hace con EPF además de los elementos vSPEM. A continuación se va a crear un punto de variación de Actividad, para ello elegiremos en el menú desplegable vp Activity y se nos abrirá una ventana como la siguiente figura:

En esta ventana contiene los siguientes campos:

•Name:

Nombre que se le dará al punto de variación.

•Description:

Una breve descripción de porque se genera dicho punto de variación.

•Icon:

El icono por defecto representativo para ese punto de variación.

•Minimum Cardinality:

Cardinalidad mínima de variantes que pueden ocupar el punto de variación.

•Maximum Cardinality:

Cardinalidad máxima de variantes que pueden ocupar el punto de variación.

•IsOpen:

Indica si el punto de variación es abierto.

Si es cerrado solo podrá ser ocupado por aquellas variantes a las que sea compatible y con las que tenga dependencias inclusividad.

Si es abierto entonces podrá ser ocupado por cualquier variante compatible.

•IsImplicit:

Por defecto no será implícito ya que es definido por el usuario, se deja para futuras versión la automatización de la definición de los puntos de variación.

•Dependencies with variants(Inclusive):

Dependencias de inclusividad con otras variantes, si añadimos variantes aquí querrá decir que el punto de variación necesitará de estas variantes individualmente para crear un elemento completo (en el caso de que sea cerrado).

•Dependencies with var points(Inclusive):

Dependencias de inclusividad con otros puntos de variación, si añadimos puntos de variación aquí querrá decir que crear una ocupación con este punto de variación en una adaptación dará como resultado que tengamos que añadir los puntos de variación añadidos en esta lista en dicha adaptación. (Las dependencias se verán más profundamente.)

•Dependencies with var points(Exclusives):

Dependencias de exclusividad con otros puntos de variación, si añadimos puntos de variación en esta lista querrá decir que provocaremos conflictos si añadimos el punto de variación creado y además alguno de los de la lista añadidos.
